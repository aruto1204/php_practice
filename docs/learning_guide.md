# 学習ガイド

このドキュメントでは、PHP学習プロジェクトの効果的な学習方法と進め方について説明します。

---

## 学習の進め方

### 1. 基本的な学習サイクル

```
理論学習 → コード実装 → 実行確認 → テスト作成 → レビュー → 次へ
```

各トピックで以下のステップを踏むことを推奨します:

#### ステップ1: 理論学習（20%）
- 各トピックの概念を理解する
- 公式ドキュメントを読む
- なぜその機能が必要なのかを考える

#### ステップ2: サンプルコードの写経（30%）
- 提供されたサンプルコードを実際に書く
- コピー&ペーストではなく、手で入力する
- コードの動きを確認しながら進める

#### ステップ3: 演習課題の実装（30%）
- 自分で考えながらコードを書く
- エラーが出たら、自分で調べて解決を試みる
- 複数の実装方法を試してみる

#### ステップ4: コードレビューとリファクタリング（20%）
- 自分のコードを見直す
- ベストプラクティスと比較する
- より良い書き方を探す

---

## 学習のコツ

### ✅ 推奨する学習方法

1. **毎日コードを書く**
   - 1日最低30分、できれば1-2時間
   - 連続性が重要（週末だけ集中より毎日少しずつ）

2. **手を動かす**
   - 読むだけでなく、必ず実装する
   - エラーを恐れずに実験する
   - トライアンドエラーを繰り返す

3. **小さく始める**
   - 最初から完璧を目指さない
   - 動くコードをまず書く
   - その後、改善していく

4. **アウトプットする**
   - 学んだことをメモする
   - ブログや SNS で共有する（オプション）
   - コードにコメントを残す

5. **公式ドキュメントを読む**
   - 信頼できる情報源を使う
   - PHP公式マニュアルを参照する習慣をつける

### ❌ 避けるべき学習方法

1. **コピー&ペーストに頼る**
   - 理解せずにコードをコピーしない
   - 必ず自分で入力する

2. **エラーをすぐに諦める**
   - エラーメッセージをよく読む
   - 検索して自分で解決を試みる

3. **一度に多くを学ぼうとする**
   - 一つずつ確実に理解する
   - 次のトピックに急がない

4. **完璧主義**
   - 最初から完璧なコードは書けない
   - まずは動くコード、その後改善

5. **孤立して学ぶ**
   - わからないことは質問する
   - コミュニティを活用する

## 各フェーズの詳細ガイド

### Phase 1: 基礎編（2-3週間）

#### 目標
- PHPの基本構文を理解し、使えるようになる
- 簡単なプログラムを自分で書けるようになる

#### 学習内容
- 変数、データ型
- 演算子
- 制御構造（if, switch, for, foreach）
- 関数
- 配列

#### 学習のポイント
- **型を意識する**: PHP 8では型宣言が重要
- **var_dump() を活用**: 変数の中身を確認する習慣をつける
- **エラーを読む**: エラーメッセージから学ぶ

#### 実践課題例
```php
// 1. 変数と型
$name = "太郎";
$age = 25;
$height = 175.5;
$isStudent = true;

// 2. 関数と型宣言
function calculateBMI(float $weight, float $height): float
{
    return $weight / (($height / 100) ** 2);
}

// 3. 配列操作
$fruits = ['りんご', 'バナナ', 'オレンジ'];
foreach ($fruits as $fruit) {
    echo $fruit . PHP_EOL;
}
```

#### チェックポイント
- [ ] 型宣言を使った関数が書ける
- [ ] 配列を操作できる
- [ ] for/foreach ループが使える
- [ ] 条件分岐が適切に使える

---

### Phase 2: 中級編（3-4週間）

#### 目標
- オブジェクト指向プログラミングを理解する
- クラスとオブジェクトを使いこなす
- ファイル操作とエラーハンドリングができる

#### 学習内容
- クラスとオブジェクト
- 継承、ポリモーフィズム
- 名前空間
- 例外処理
- ファイル操作

#### 学習のポイント
- **OOPの考え方を理解**: なぜOOPが必要なのか
- **実際のコードで練習**: 具体的な例で実装
- **PSR-4を意識**: オートローディングの理解

#### 実践課題例
```php
namespace App\Models;

class User
{
    public function __construct(
        private string $name,
        private string $email,
        private int $age
    ) {}

    public function getName(): string
    {
        return $this->name;
    }

    public function isAdult(): bool
    {
        return $this->age >= 18;
    }
}

// 使用例
$user = new User('太郎', 'taro@example.com', 25);
echo $user->getName(); // 太郎
```

#### チェックポイント
- [ ] クラスを設計できる
- [ ] 継承を適切に使える
- [ ] 名前空間を理解している
- [ ] 例外処理を実装できる
- [ ] ファイル操作ができる

### Phase 3: 応用編（4-5週間）

#### 目標
- データベースと連携したアプリケーションを作る
- セキュリティを考慮したコードが書ける
- REST APIの基礎を理解する

#### 学習内容
- PDOによるデータベース操作
- セキュリティ対策
- フォーム処理
- バリデーション
- REST API

#### 学習のポイント
- **セキュリティファースト**: 常にセキュリティを意識
- **プリペアドステートメント**: SQLインジェクション対策
- **入力バリデーション**: ユーザー入力を信頼しない

#### 実践課題例
```php
// データベース接続
$pdo = new PDO(
    'mysql:host=localhost;dbname=php_practice;charset=utf8mb4',
    'phpuser',
    'password',
    [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]
);

// プリペアドステートメント
$stmt = $pdo->prepare('SELECT * FROM users WHERE email = :email');
$stmt->execute(['email' => $email]);
$user = $stmt->fetch();

// セキュアなパスワード処理
$hashedPassword = password_hash($password, PASSWORD_DEFAULT);
if (password_verify($inputPassword, $hashedPassword)) {
    // ログイン成功
}
```

#### チェックポイント
- [ ] データベースCRUD操作ができる
- [ ] SQLインジェクション対策ができる
- [ ] XSS対策ができる
- [ ] CSRF対策を理解している
- [ ] 簡単なREST APIが作れる

## デバッグのコツ

### 1. var_dump() と print_r()

```php
// 変数の中身を確認
var_dump($variable);

// 配列の構造を見やすく表示
print_r($array);

// die() と組み合わせて実行を止める
var_dump($data);
die();
```

### 2. エラーメッセージを読む

```php
// エラー表示を有効化（開発環境のみ）
ini_set('display_errors', '1');
error_reporting(E_ALL);

// エラーログを確認
tail -f /var/log/php/error.log
```

### 3. Xdebug を使う

- ブレークポイントを設定
- ステップ実行で動きを確認
- 変数の値を監視

## 質問・相談の仕方

### 良い質問の例

```
【質問】PDOでデータベースに接続できない

【環境】
- PHP 8.3
- MySQL 8.0
- macOS

【やったこと】
1. PDOオブジェクトを作成
2. 接続を試みる

【エラーメッセージ】
SQLSTATE[HY000] [2002] Connection refused

【試したこと】
- MySQL が起動していることを確認
- ポート番号が正しいか確認

【コード】
```php
$pdo = new PDO(
    'mysql:host=localhost;dbname=test',
    'root',
    'password'
);
```

### 避けるべき質問

- 「動きません」だけの質問
- エラーメッセージを添付しない
- 環境情報がない
- 自分で調べた形跡がない

## よくある質問

### Q: どのくらいのペースで学習すべきですか？

A: 個人のペースでOKですが、推奨は週7-10時間です。毎日1-2時間の学習が理想的です。

### Q: わからないことがあったらどうすればいいですか？

A: まず公式ドキュメントを確認し、それでもわからなければStack OverflowやQiitaで検索してください。

### Q: 全てを完璧に理解してから次に進むべきですか？

A: いいえ。70-80%理解できたら次に進み、実践で学ぶことも重要です。

### Q: フレームワークはいつ学ぶべきですか？

A: Phase 4を完了し、基礎がしっかりしてからLaravelやSymfonyを学ぶことを推奨します。
