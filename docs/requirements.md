# PHP 学習プロジェクト 要件定義書

## 1. プロジェクト概要

### 1.1 目的

PHP の基礎から実践的なスキルまで体系的に習得できる学習教材を作成する。
現代の PHP ベストプラクティスに基づいた実践的なコーディングスキルの獲得を目指す。

### 1.2 対象者

- PHP を初めて学ぶプログラミング初心者
- 他言語の経験があり、PHP を学びたい開発者
- 古い PHP の知識をモダンな手法にアップデートしたい開発者

## 2. 学習目標

### 2.1 基礎知識

- PHP の基本構文と文法の理解
- 変数、データ型、演算子の使い方
- 制御構造（条件分岐、繰り返し）の理解
- 関数の定義と使用方法
- 配列とその操作方法

### 2.2 中級知識

- オブジェクト指向プログラミング（OOP）
  - クラスとオブジェクト
  - 継承、カプセル化、ポリモーフィズム
  - トレイトとインターフェース
- 名前空間とオートローディング
- エラーハンドリングと例外処理
- ファイル操作とストリーム
- セッションとクッキーの管理

### 2.3 実践的スキル

- データベース操作（PDO 使用）
- セキュリティベストプラクティス
  - SQL インジェクション対策
  - XSS 対策
  - CSRF 対策
  - パスワードハッシュ化
- フォーム処理とバリデーション
- RESTful API の設計と実装
- Composer による依存関係管理
- PSR スタンダードの理解と適用

### 2.4 モダン PHP の実践

- PHP 8.x の新機能
  - 名前付き引数
  - Union 型と Mixed 型
  - Match 式
  - コンストラクタプロモーション
  - アトリビュート
- 型宣言（Type Hints）の活用
- デザインパターンの実装
- テスト駆動開発（PHPUnit）
- フレームワークの基礎理解

## 3. 学習コンテンツ構成

### 3.1 カリキュラム構造

```
01_basics/              # 基礎編
  - variables.php       # 変数とデータ型
  - operators.php       # 演算子
  - control_flow.php    # 制御構造
  - functions.php       # 関数
  - arrays.php          # 配列

02_intermediate/        # 中級編
  - oop_basics/         # OOP基礎
  - namespaces/         # 名前空間
  - error_handling/     # エラー処理
  - file_operations/    # ファイル操作
  - sessions/           # セッション管理

03_advanced/            # 応用編
  - database/           # データベース操作
  - security/           # セキュリティ
  - validation/         # バリデーション
  - api/                # API開発

04_modern_php/          # モダンPHP
  - php8_features/      # PHP8の新機能
  - design_patterns/    # デザインパターン
  - testing/            # テスト
  - composer/           # Composer

05_projects/            # 実践プロジェクト
  - todo_app/           # Todoアプリ
  - blog_system/        # ブログシステム
  - rest_api/           # REST API
```

### 3.2 各レッスンの構成要素

- **概念説明**: 各トピックの理論的背景
- **コード例**: 実装可能なサンプルコード
- **実践演習**: 学習者が実際に書くコード課題
- **ベストプラクティス**: 推奨される実装方法
- **アンチパターン**: 避けるべきコーディング例
- **参考資料**: 公式ドキュメントや関連リソース

## 4. 技術要件

### 4.1 必須環境

- PHP 8.1 以上
- Composer 2.x
- SQLite 3.x（PHP組み込み）
- Git

### 4.1.1 オプション環境

- MySQL 8.0以上（Phase 3以降で使用）
- PostgreSQL 13以上（代替オプション）

### 4.2 推奨開発環境

- Visual Studio Code / PHPStorm
- Xdebug（デバッグツール）
- Docker（環境構築用）
- PHPStan / Psalm（静的解析ツール）

### 4.3 コーディング規約

- PSR-1: Basic Coding Standard
- PSR-4: Autoloading Standard
- PSR-12: Extended Coding Style
- 型宣言の積極的な使用
- ドキュメンテーションコメント（PHPDoc）

## 5. ベストプラクティス

### 5.1 セキュリティ

- ユーザー入力は常にサニタイズとバリデーション
- プリペアドステートメントの使用
- パスワードは`password_hash()`で暗号化
- HTTPS 通信の推奨
- セキュリティヘッダーの設定

### 5.2 コード品質

- DRY 原則（Don't Repeat Yourself）
- SOLID 原則の適用
- 適切な命名規則
- 関数は単一責任を持つ
- マジックナンバーの回避

### 5.3 パフォーマンス

- OPcache の有効化
- 不要なクエリの削減
- 適切なインデックス設計
- キャッシング戦略の実装

### 5.4 保守性

- コードコメントの適切な使用
- テストコードの作成
- バージョン管理の活用
- ログ記録の実装

## 7. 学習ロードマップ

### Phase 1: 基礎固め（2-3 週間）

- PHP 基本構文の習得
- 簡単なプログラムの作成
- 開発環境のセットアップ

### Phase 2: 中級スキル（3-4 週間）

- OOP の理解と実践
- データベース連携
- エラーハンドリング

### Phase 3: 実践スキル（4-5 週間）

- セキュリティ対策の実装
- API 開発
- モダン PHP 機能の活用

### Phase 4: プロジェクト開発（3-4 週間）

- 総合的な Web アプリケーション開発
- コードレビューとリファクタリング
- デプロイメント
